Fixed issue where the account totaller wasn't representing the full total. This happened because the totaller assumed categories were unique. With the introduction of years, a single category name may be present multiple times (e.g. 2023 -> Maint., 2024 -> Maint.). The structure used to keep totals was a map of [category] -> total, so only the last written category for whatever year was being captured. The straightforward solution was to change the structure to [year] -> { [category] -> total }.
